## Parallel workflow

- Run **several Cursor agents** (or humans) at once: assign **one batch file per agent**.
- Typical batch size: **200** papers per pass (~7 passes for a full corpus of ~1278 pages).
- After a batch is curated, remove `phase4-stub-pending-curation` from `candidate_tags` and replace stub prose; then regenerate batches so finished work drops out of `--mode pending`.

## Regenerate

```bash
python3 scripts/paper_curation_batches.py --batch-size 200 --mode pending
```

Full list (including already curated pages), e.g. for a second editorial pass:

```bash
python3 scripts/paper_curation_batches.py --batch-size 200 --mode all
```

See `docs/PHASE4_RUNBOOK.md` for Phase 4 policy.
